## Why

The `claude-spec-driven` schema uses a single-agent, sequential execution model during the apply phase. Claude Code now supports multi-agent orchestration via `Team`, isolated parallel work via `WorkTree`, and concurrent execution via `/batch`. The schema should be optimized so that the apply phase uses role-based parallelism — spawning teammates for implementation, tests, and documentation that work concurrently in a single worktree, then challenge each other's work.

## What Changes

- **BREAKING** Rewrite apply-phase instructions to use a step-sequential, role-parallel Team model: tasks grouped by incremental visible step, within each step split by concern (implementation, tests, docs), with cross-challenge after each step
- Add a "Parallelization Strategy" section to the design template that identifies independent concerns and conflict zones
- Update the tasks template to organize tasks by incremental step with role subgroups (implementation, tests, docs) within each step
- Update proposal template Impact section to capture concurrency considerations

## Capabilities

### New Capabilities

- `role-based-apply`: Defines how the apply phase spawns a Team with role-specific teammates (implementer, tester, documenter) that work in parallel within a single worktree, then cross-review each other's output
- `cross-challenge-phase`: Defines the review protocol where teammates validate each other's work — tests verify implementation, docs match behavior, implementer reviews test coverage

### Modified Capabilities

_None — this is a new schema fork with no existing specs._

## Impact

- `schema.yaml`: Apply section rewritten for role-based team orchestration; tasks artifact instruction updated for role-grouped format
- `templates/tasks.md`: Restructured with role-based groupings (Implementation, Tests, Documentation)
- `templates/design.md`: New "Parallelization Strategy" section added
- `templates/proposal.md`: Impact section expanded to cover concurrency considerations
- Downstream effect: Changes generated by this schema will produce role-grouped task lists that Claude Code can execute with Team-based parallelism
